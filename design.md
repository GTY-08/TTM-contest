# 운동 틈틈 (Teumteum) — Design Language

## Overview

운동 틈틈의 인터페이스는 **따뜻하고 부담 없는 "동네 운동 메이트" 표면**이다 — 부드러운 크림색 캔버스 위에 흰색 라운드 카드가 잔잔하게 놓이고, 단 하나의 자신감 있는 그린이 "이건 중요해 / 여기를 눌러"의 역할을 모두 맡는다. 제품 전체가 마켓플레이스라기보다 친근한 동네 게시판처럼 느껴진다: 무엇도 시끄럽지 않고, 무엇도 날카롭지 않으며, 따뜻한 오프화이트 배경이 긴 세션에도 눈을 편하게 해 준다.

밀도는 의도적으로 낮다. 각 화면은 크림 위에 떠 있는 흰색 카드들의 세로 스택이며, 넉넉한 공기와 조용한 회색 섹션 라벨(`신뢰 정보`, `활동 포인트`, `활동 기록`)로 구분된다. 하드 보더도, 무거운 아웃라인도, 경쟁하는 액센트 컬러도 없다. 입체감은 오직 한 가지, **부드럽고 넓게 퍼지는 낮은 불투명도의 드롭 섀도**에서 온다. 이 섀도가 각 흰색 카드를 따뜻한 종이 위에 놓인 실제 카드처럼 보이게 한다.

시그니처는 **크림 위 흰색 카드 리듬**이다: 카드는 항상 순백, 배경은 항상 따뜻한 크림, 그리고 그 밝기 차이 — 보더가 아니라 — 가 콘텐츠를 캔버스에서 분리한다. 단일 그린(`{colors.primary}`)이 모든 상호작용·신뢰 신호를 담당한다: 로고, 활성 탭, 토글, 스탯 하이라이트, 별점, 주요 CTA, 인증 배지. 그 밖의 모든 것은 차분한 near-black 잉크이거나 부드러운 웜 그레이다.

**핵심 특성**

- 따뜻한 크림 캔버스(`{colors.canvas}`) + 순백 카드 — 밝기 차이가 구분선이지, 보더가 아니다.
- 하나의 그린 액센트(`{colors.primary}`)가 모든 상호작용·신뢰·강조 신호를 담당. 두 번째 브랜드 컬러는 없다.
- 크고 친근한 코너 반경(카드 20–28px) — 부드럽고 다가가기 쉬우며, 결코 날카롭지 않다.
- 정확히 하나의 입체 처리: 부드럽고 넓은 카드 섀도. 보더도, 이너 섀도도, 그라데이션도 없다.
- 흰색 카드 그룹 위에 쌓인 조용한 회색 섹션 라벨이 크롬 없이 구조를 준다.
- 전체 Pretendard; 숫자·헤딩은 볼드(700–800), 본문은 레귤러, 보조는 소프트 그레이.
- 그린 틴트 원형 아이콘 배지(`{colors.surface-tint}`)가 각 카드의 목적에 아웃라인 글리프를 짝지어 준다.
- 알약형 상태 칩("인증 완료", "모집 중")이 소프트 그린 틴트로 상태를 차분히 전달한다.
- 빈 상태는 일급 시민이다: 친근한 문장 + 부드러운 안내, 결코 텅 빈 공백이 아니다.

## Colors

> **분석한 소스 화면:** 홈(Home), 프로필(Profile). 시스템은 화면 전반에서 동일하며 카드 콘텐츠만 바뀐다. 긴 세션의 눈 피로를 줄이기 위해 캔버스를 최근에 밝게 조정했다 — 크림은 이제 wash가 아니라 whisper다.

### Brand & Accent

- **Teum Green** (`{colors.primary}` — `#2EA86A`): 유일한 브랜드 레벨 상호작용 컬러. 로고 워드마크, 활성 탭, 토글 ON 상태, 스탯 하이라이트 숫자, 별점, 주요 CTA 텍스트, 홈의 섹션 타이틀("주변 레이드"), 그리고 "+" 액션 글리프. 이것이 유일한 "이건 중요해 / 여기를 눌러" 신호다.
- **Teum Green Deep** (`{colors.primary-strong}` — `#15803F`): 그린 틴트 표면 위에 앉는 텍스트/아이콘(CTA 카드 타이틀, 배지 글리프)에 사용 — mid-green이면 대비를 잃는 자리.
- **Toggle Green** (`{colors.primary-bright}` — `#34C759`): 활성 토글 트랙 전용의 더 밝고 채도 높은 그린 — 의도적으로 "살아 있는" 그린으로, 켜진 스위치처럼 읽힌다.

### Surface

- **Warm Cream** (`{colors.canvas}` — `#FBFAF4`): 페이지 배경. 밝게 조정된 저채도 웜 오프화이트 — 따뜻하고 비임상적으로 느껴질 만큼은 존재하되, 긴 세션에 눈을 피로하게 하지 않을 만큼 밝다. 새롭게 부드러워진 캔버스.
- **Pure White** (`{colors.surface-card}` — `#FFFFFF`): 모든 콘텐츠 카드, 스탯 블록, 리스트 행, 그리고 하단 탭 바. 크림 대비의 밝기 차이가 분리 시스템 전부다.
- **Green Tint** (`{colors.surface-tint}` — `#E6F0E9`): 원형 아이콘 배지(본인 인증 shield, 활동 포인트 reward, 레이드 열기 "+")와 주요 CTA 카드 배경의 채움. 그린 액센트의 차분한 저채도 동반자.
- **Green Tint Soft** (`{colors.surface-tint-soft}` — `#EEF4EF`): 중립/긍정 상태의 알약 칩("인증 완료", "모집 중") — 배지 틴트보다 한 단계 더 부드럽다.
- **Tab Active Tint** (`{colors.surface-tab-active}` — `#DDEEE3`): 활성 하단 탭 아이콘 뒤의 라운드 알약 하이라이트.

### Text

- **Ink** (`{colors.ink}` — `#1E2A23`): 헤드라인, 카드 타이틀, 이름, 큰 스탯 숫자. 순수 블랙이 아니라 옅은 웜-그린 언더톤의 near-black — UI를 인쇄물이 아니라 부드럽고 온-브랜드로 유지한다.
- **Body** (`{colors.body}` — `#3D423C`): 흰색 표면 위 기본 문단·카드 본문.
- **Muted** (`{colors.muted}` — `#9AA0A0`): 섹션 라벨(`신뢰 정보`, `활동 포인트`), 보조 캡션, 날짜, 타이틀 하위 카피, 비활성 탭 라벨. 조용한 일꾼 그레이.
- **Muted Soft** (`{colors.muted-soft}` — `#B8BDB8`): 플레이스홀더 / 빈 상태 보조 텍스트, 옅은 헬퍼 라인.
- **On Green** (`{colors.on-primary}` — `#FFFFFF`): 토글 노브, 그리고 솔리드 그린 채움 위에 앉는 모든 텍스트/아이콘.

### Hairlines & Dividers

- **Card Divider** (`{colors.divider}` — `#EFECE3`): 하나의 카드 안에서 행을 구분하는 옅은 인셋 라인(예: 본인 인증 → 활동 등급, 또는 스탯 컬럼 사이 세로 룰). 하드 룰이 아니라 whisper로 기능한다.

### Brand Gradient

**장식용 그라데이션 없음.** 따뜻함은 크림 캔버스 톤 자체와 부드러운 카드 섀도에서 오지, CSS 그라데이션에서 오지 않는다. 시스템에서 그라데이션에 가장 근접한 것은 사용자 아바타의 사진 콘텐츠뿐이다. 틈틈은 의도적으로 플랫 컬러 시스템이다 — 깊이는 섀도 + 표면 밝기이지, 결코 그라데이션 채움이 아니다.

## Typography

### Font Family

- **모든 텍스트**: `Pretendard, -apple-system, system-ui, sans-serif` — 모든 굵기·크기에 걸쳐 쓰이는 단일 한국어 최적화 지오메트릭 산세리프. Pretendard의 고른 한글 리듬과 촘촘한 라틴 메트릭이 제품 전체를 이끈다; display/text 분리는 없다.

### Hierarchy

| Token                        | Size | Weight | Line Height | Use                                                                              |
| ---------------------------- | ---- | ------ | ----------- | -------------------------------------------------------------------------------- |
| `{typography.greeting}`      | 23px | 800    | 1.3         | 홈 인사 헤드라인("안녕하세요, …님")                                               |
| `{typography.screen-title}`  | 22px | 800    | 1.3         | 화면 타이틀(프로필), 홈의 섹션 타이틀(주변 레이드) — `{colors.primary}`로 설정    |
| `{typography.stat-number}`   | 26px | 800    | 1.1         | 큰 활동 숫자("3회", "4회", "3개")                                                 |
| `{typography.card-title}`    | 17px | 700    | 1.35        | 카드 타이틀(활동 포인트 관리, 레이드 열기, 본인 인증)                             |
| `{typography.list-title}`    | 15px | 700    | 1.4         | 리스트 행 타이틀, CTA 카드 타이틀                                                 |

| `{typography.stat-caption}`  | 13px | 500    | 1.3         | 숫자 아래 스탯 라벨(모집한 레이드, 참여한 레이드)                                 |
| `{typography.caption}`       | 13px | 400    | 1.4         | 날짜, 하위 카피, 헬퍼 텍스트                                                      |
| `{typography.chip}`          | 12px | 700    | 1.0         | 상태 알약 칩(인증 완료, 모집 중)                                                  |
| `{typography.tab-label}`     | 11px | 500    | 1.0         | 하단 탭 라벨                                                                      |

### Principles

- **"단단하게" 느껴져야 할 것은 Weight 800.** 인사, 화면 타이틀, 큰 스탯 숫자는 모두 800에 앉는다 — 볼드 굵기가 틈틈이 "이건 진짜, 자리 잡은 데이터"라고 신호하는 방식이며, 비어 보이는/프로토타입 느낌과 정면으로 싸운다.
- **카드 타이틀은 700, 본문은 400.** 사다리는 400 / 500 / 700 / 800 — 깔끔하고 제한적이다. 600은 의도적으로 건너뛰어 타이틀과 본문의 대비를 명확히 한다.
- **그린은 채움만이 아니라 타입 컬러다.** 홈의 섹션 타이틀과 화면 타이틀은 `{colors.primary}`를 텍스트 컬러로 쓴다 — 그린은 버튼만이 아니라 위계를 담당한다.
- **회색 섹션 라벨이 구조 작업을 한다.** 구분선이나 박스 대신 조용한 14px/700 회색 라벨(`{colors.muted}`)이 각 카드 그룹을 소개한다. 이 라벨이 섹션 사이의 유일한 "크롬"이다.
- **데이터 카드에서는 숫자가 주인공.** 스탯 숫자는 오버사이즈(26px/800)로 중앙 정렬되고, 캡션은 그 아래 작고 회색으로 — 눈은 값을 먼저, 의미를 나중에 잡는다.

## Layout

### Spacing System

- **기본 단위:** 4px. 구조 레이아웃은 8 / 12 / 16 / 20 / 24로 스냅된다.
- **토큰:** `{spacing.xs}` 8px · `{spacing.sm}` 12px · `{spacing.md}` 16px · `{spacing.lg}` 20px · `{spacing.xl}` 24px · `{spacing.section}` 28px.
- **화면 좌우 패딩:** 양 끝 20px — 카드는 결코 화면 가장자리에 닿지 않고, 크림이 항상 카드를 감싼다.
- **섹션 간격:** 카드 그룹과 다음 회색 섹션 라벨 사이 `{spacing.section}`(28px).
- **라벨-카드 간격:** 회색 섹션 라벨과 그것이 소개하는 카드 사이 12px.
- **카드 내부 패딩:** 18–20px.
- **리스트 내 형제 카드 간격:** 10–12px.

### Grid & Container

- **단일 컬럼 스택.** 모든 화면은 풀-width 카드의 세로 컬럼 하나다; 스탯 카드 내부를 제외하면 멀티 컬럼 콘텐츠 그리드는 없다.
- **스탯 카드:** 하나의 흰색 카드 안에서 동일한 3-또는 4-컬럼 분할, 컬럼은 옅은 세로 `{colors.divider}` 룰로 구분되고 각 컬럼은 중앙 정렬(숫자 위 캡션).
- **카드 너비:** 풀 콘텐츠 너비(화면에서 20px 거터 제외).

### Whitespace Philosophy

크림이 숨 쉴 공간이다. 카드는 넉넉히 패딩되고 결코 붐비지 않는다; 빈 화면(주변 레이드 없음)은 콘텐츠를 늘여 채우기보다 차분함을 유지한다 — 하단에 남은 크림은 의도적이며, "미완성"이 아니라 "대기 중인 것 없음, 모두 정리됨"을 신호한다. 밀도가 오르는 유일한 곳은 데이터 카드 내부로, 스탯이 어깨를 맞대고 앉아 한눈에 읽히는 단일 유닛으로 읽힌다.

## Elevation & Depth

| Level         | Treatment                                                    | Use                                                    |
| ------------- | ------------------------------------------------------------ | ------------------------------------------------------ |
| Flat          | 섀도 없음, 보더 없음                                         | 크림 캔버스, 섹션 라벨, 인라인 텍스트                  |
| Card          | `0 2px 10px rgba(30, 50, 35, 0.05)` — 부드럽고 넓은 저불투명 | 모든 흰색 카드, 스탯 블록, 리스트 행, 탭 바            |
| Inset divider | 1px `{colors.divider}` 라인                                  | 카드 _내부_의 행 구분선과 스탯 컬럼 룰                 |

**섀도 철학.** 틈틈은 **정확히 하나의** 입체를 쓴다: 흰색 카드 아래에 깔리는 부드럽고 넓은, 거의 보이지 않는 섀도가 카드를 크림에서 들어 올린다. 결코 하드 드롭으로 조여지지 않고, 칩·글리프·텍스트에 적용되지 않으며, 이중으로 겹쳐지지 않는다. UI의 깊이는 (a) 흰색-온-크림 밝기 차이와 (b) 이 단일 부드러운 섀도다. 그 외에는 아무것도 그림자를 드리우지 않는다.

### Decorative Depth

- **표면 밝기 차이**(크림 위 흰색 카드)가 주요 구분선 — 카드 어디에도 보더 없음.
- **그린 틴트 원형 배지**가 크롬이나 섀도를 더하지 않고 각 카드에 작은 초점 "오브젝트"를 준다.
- **부드러운 카드 섀도**가 쌓인 종이 카드로 읽힐 만큼의 물리성만 공급한다.

## Shapes

### Border Radius Scale

| Token               | Value  | Use                                             |
| ------------------- | ------ | ----------------------------------------------- |
| `{rounded.chip}`    | 9999px | 상태 알약 칩, 토글 트랙, 활성 탭 알약 하이라이트 |
| `{rounded.badge}`   | 50%    | 원형 아이콘 배지, 아바타, 알림 점                |
| `{rounded.card-sm}` | 18px   | 리스트 행(후기 카드, 레이드 카드)                |
| `{rounded.card}`    | 22px   | 표준 콘텐츠 카드(신뢰, 포인트, CTA)              |
| `{rounded.card-lg}` | 26px   | 큰 스탯 카드와 토글/상태 카드                    |

### Geometry

- **모든 것이 둥글다.** 가장 작은 상호작용 요소는 완전한 알약/원이고, 카드는 18–26px에 앉는다. 제품 어디에도 날카로운 코너는 없다 — 부드러움은 브랜드 가치다.
- **아이콘 배지**는 `{colors.surface-tint}`의 완전한 원(`{rounded.badge}`), ~52px, `{colors.primary-strong}`의 24px 아웃라인 글리프를 담는다.
- **상태 칩**은 `{colors.surface-tint-soft}`의 완전한 알약(`{rounded.chip}`)에 12px/700 green-deep 텍스트.
- **아바타**는 원형 사진 크롭, ~40px, 홈 우상단.

## Components

### Top Bars

**`home-header`** — 홈의 상단 행. 왼쪽: 그린 앱 글리프 + `{colors.primary}`의 "틈틈" 워드마크(21px/800). 오른쪽 클러스터: 아웃라인 알림 벨(작은 그린 미확인 점) + 원형 아바타. 배경 채움 없음 — 크림 위에 바로 앉는다. 그 아래: 인사 헤드라인 + 작은 회색 서브라인.

**`screen-header`** — 내부 화면 헤더(프로필). 왼쪽: `{colors.primary}`의 화면 타이틀(22px/800). 오른쪽: `{colors.body}`의 조용한 아웃라인 액션 글리프(설정 기어) 하나. 크림 위에 플랫, 아래 구분선 없음.

### Cards

**`section-label`** — 카드가 아니라 연결 조직: 각 카드 그룹 위 12px에 앉는 회색(`{colors.muted}`) 14px/700 라벨(신뢰 정보 / 활동 포인트 / 활동 기록 / 받은 후기). 크롬 없이 모든 구조적 그룹핑을 제공한다.

**`info-card`** — 하나 이상의 행을 담는 흰색 카드(`{rounded.card}`, 22px). 각 행: 원형 그린 틴트 아이콘 배지(왼쪽) + `{typography.card-title}` 타이틀 + 우측 정렬 상태 알약 칩. 여러 행은 인셋 `{colors.divider}` 라인으로 구분(예: 본인 인증 / 활동 등급). 부드러운 카드 섀도, 보더 없음.

**`nav-card`** — 내비게이션용 탭 가능 흰색 카드(활동 포인트 관리). 원형 그린 틴트 배지 + 타이틀 + 회색 서브 카피(왼쪽); 오른쪽에 `{colors.muted}`의 조용한 셰브런(`›`). `{component.info-card}`와 동일한 반경/섀도.

**`stat-card`** — 옅은 세로 `{colors.divider}` 룰로 동일하게 중앙 정렬 컬럼으로 나뉜 흰색 카드(`{rounded.card-lg}`, 26px). 각 컬럼: `{typography.stat-number}`(`{colors.ink}`)의 큰 숫자 위 작은 회색 `{typography.stat-caption}`. 홈에서는 강조/포인트 숫자가 `{colors.primary}`로 렌더된다.

**`status-toggle-card`** — 홈의 활동 카드. 왼쪽: 그린 상태 점 + `{colors.primary}`의 "활동 중"(15px/700) 위 회색 서브라인. 오른쪽: 완전한 알약 토글 — ON일 때 트랙 `{colors.primary-bright}`, 흰색 노브(`{colors.on-primary}`)가 오른쪽으로 밀림. `{rounded.card-lg}`.

**`cta-card`** — 주요 "레이드 열기" 액션. 흰색이 아니라 `{colors.surface-tint}` 배경의 풀-width 카드로 스타일된다. 왼쪽: 그린 "+" 글리프를 담은 큰 원형 배지. 가운데: 타이틀(`{colors.primary-strong}`, 17px/800) + 그린-그레이 서브 카피. 오른쪽: 그린 셰브런. 이 틴트 카드가 "해야 할 메인 작업"으로 읽히는 유일한 섹션이다.

**`review-row`** — 흰색 리스트 카드(`{rounded.card-sm}`, 18px). 상단 행: 그린 별 + 평점 숫자(왼쪽), `{colors.muted}`의 날짜(오른쪽). 아래: `{typography.body}`의 후기 텍스트; 빈 상태("작성된 후기는 없어요.")는 이탤릭 `{colors.muted-soft}`로 렌더된다.

**`empty-state-card`** — 없는 콘텐츠의 자리를 친근한 타이틀(`{typography.list-title}`) + 부드러운 회색 안내 라인으로 채우는 흰색 카드. 결코 빈 영역이나 스피너가 아니다 — 차분한 문장 _자체_가 상태다.

### Chips & Badges

**`status-chip`** — `{colors.surface-tint-soft}`의 완전한 알약 칩, `{colors.primary-strong}` 12px/700 텍스트("인증 완료", "모집 중"). 긍정/중립 상태를 조용히 전달; 아이콘 불필요.

**`icon-badge`** — `{colors.primary-strong}`의 24px 아웃라인 글리프를 담은 원형 `{colors.surface-tint}` 디스크(~52px). 모든 카드에 반복되는 시각적 앵커.

**`notification-dot`** — 크림 링을 두른 작은 `{colors.primary}` 원, 벨 아이콘 우상단에 고정.

### Bottom Navigation

**`tab-bar`** — 부드러운 카드 섀도와 넉넉한 세이프-에어리어 하단 패드를 두고 바닥에 고정된 흰색 바(`{colors.surface-card}`). 다섯 항목: 홈 · 모집 · 찾기 · 활동 · 프로필. 각각 아웃라인 글리프 + 라벨. 활성 항목의 아이콘은 `{colors.surface-tab-active}` 라운드-알약 하이라이트 안에 앉고 아이콘 + 라벨은 `{colors.primary}`; 비활성 항목은 `{colors.muted}` 글리프 + 라벨(`{typography.tab-label}`, 11px/500). "모집" 항목은 생성 액션으로 읽히도록 원 안의 "+"를 쓴다.

## Do's and Don'ts

### Do

- 모든 상호작용·신뢰·강조 신호에 `{colors.primary}`(Teum Green)를 쓴다 — 로고, 활성 탭, 토글, 스탯 하이라이트, 별, CTA, 배지 — 그리고 오직 그것만. 단일 액센트가 브랜드다.
- 카드는 따뜻한 크림 캔버스 위에서 순백으로 유지하고, **밝기 차이**를 구분선으로 삼는다. 카드에 보더 없음.
- 흰색 카드에만 하나의 부드러운 카드 섀도(`0 2px 10px rgba(30,50,35,0.05)`)를 적용한다 — 칩·글리프·텍스트에는 절대 금지.
- 넉넉히 둥글린다: 카드 18–26px, 칩/토글은 완전한 알약, 배지는 원. 부드러움이 온-브랜드다.
- 각 카드 그룹을 박스나 룰 대신 조용한 회색 `{typography.section-label}`로 소개한다.
- 큰 숫자는 weight 800으로 오버사이즈한다 — 자리 잡은 볼드 데이터가 앱을 프로토타입이 아니라 진짜로 느끼게 한다.
- 빈 상태를 일급으로 다룬다: 따뜻한 문장 + 부드러운 안내로 공간을 채우기보다 차분함을 유지한다.
- 화면의 단 하나의 가장 중요한 액션은 그린 틴트 `{component.cta-card}`로 표시한다.

### Don't

- 두 번째 액센트 컬러를 도입하지 않는다 — 블루도, 레드도, 골드 보더도 없다. 모든 "눌러 / 믿어" 신호는 그린이다.
- 카드에 보더나 하드 아웃라인을 더하지 않는다; 크림-투-화이트 밝기 단차가 그것을 대체한다.
- 그라데이션을 배경으로 쓰지 않는다 — 따뜻함은 크림 톤 + 부드러운 섀도이지, 결코 그라데이션 채움이 아니다.
- 캔버스를 다시 무거운 크림 쪽으로 어둡게 하지 않는다; 밝게 조정된 웜 오프화이트는 긴 세션의 눈 피로를 줄이기 위해 존재한다.
- weight 600을 쓰지 않는다 — 사다리는 400 / 500 / 700 / 800이며, 타이틀-본문 대비를 또렷이 하려 600은 의도적으로 건너뛴다.
- "컴팩트함"을 위해 코너를 날카롭게 하거나 반경을 줄이지 않는다 — 부드럽고 친근한 지오메트리는 브랜드 가치다.
- 카드 섀도를 이중으로 하거나 하드 드롭으로 조이지 않는다; 하나의 부드럽고 넓은 섀도가 입체 시스템 전부다.
- 출시 화면에 날것/빈 데이터("○○", "0회", 깨지거나 테스트용 문자열)를 남기지 않는다 — 실제 값을 시드하거나 친근한 빈 상태를 보여준다.

## Responsive Behavior

### Frame

- ~390pt 논리 너비(390×844 레퍼런스)의 단일 모바일 컬럼용으로 설계. 콘텐츠는 20px 좌우 거터 안 풀-width 카드의 세로 스크롤.

### Touch Targets

- 최소 44×44px. 탭 바 항목, 토글, 탭 가능 카드는 모두 이를 초과한다; 토글과 아이콘 배지는 여유 있게 더 크다.
- 내비게이션하는 카드 전체가 탭 가능(`{component.nav-card}`, `{component.cta-card}`) — 넉넉한 카드 영역이 히트 타깃이다.

### Scaling

- 타입과 간격은 절대 크기를 유지한다; 작은 기기에서는 폰트가 줄기 전에 컬럼이 먼저 거터를 좁힌다.
- 하단 `{component.tab-bar}`는 상시 표시되며 기기 하단 세이프-에어리어 인셋을 존중한다.

## Iteration Guide

1. 한 번에 하나의 카드를 작업하고, 그 키를 직접 참조한다(`{component.stat-card}`, `{component.cta-card}`).
2. 상태 변형(토글 ON/OFF, 칩 긍정/중립, 탭 활성/비활성)은 새 컬러가 아니라 별도 노트로 존재한다.
3. 어디서나 `{token.refs}`를 쓴다 — 절대 hex를 인라인하지 않는다.
4. 강조 순서(헷갈릴 때): (1) 그린으로 만든다, (2) 숫자를 더 크게/굵게, (3) 틴트 CTA 카드로 옮긴다 — 절대 보더나 두 번째 컬러를 더하지 않는다.
5. 카드 = 흰색, 캔버스 = 크림, 액센트 = 그린. 이 세 상수는 깰 수 없다.
6. 하나의 부드러운 카드 섀도는 흰색 카드 전용이다.

## Known Gaps

- 눌림/활성 마이크로 인터랙션(카드 탭 피드백, 버튼 프레스 스케일)은 분석 화면에 담기지 않았다; 부드러운 브랜드 느낌에 맞춰 시스템 전역 `transform: scale(0.97)` 프레스 상태를 권장한다.
- 에러·폼 검증 상태는 아직 표면화되지 않았다; 긍정/중립 상태 칩만 문서화되어 있다.
- 다크 모드는 정의되지 않았다; 문서화된 시스템은 웜-라이트 기본값이다.
- 정확한 카드 섀도와 크림 hex는 밝게 조정된 화면에서 눈대중으로 잡은 값이며, 캔버스 톤이 확정되면 디자인 토큰으로 고정해야 한다.
- 온보딩 일러스트 컴포넌트는 제거되었다. 이후 온보딩 성격의 시각 요소가 다시 필요해지면 위와 동일한 컬러/반경/섀도 시스템을 따른다.
